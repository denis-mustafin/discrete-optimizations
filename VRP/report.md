# Решение задачи VRP

## Структура репозитория

- `main.cpp` - код текущего решения. Читает со стандартного ввода входной файл. В первой строке выводит счет решения на тесте, в каждой из следующих $v$ строк - количество людей, к которым едет очередная машина и номера этих людей в порядке, в котором их надо посещать.
- `check.cpp` - код чекера. Аргументами командной строки получает пути до входного и выходного файлов. Проверяет корректность вывода и выводит его счет.
- `main` и `check` - скомпилированные (под Линукс) версии `main.cpp` и `check.cpp`
- `scored_data` - папка с оцениваемыми тестами
- `scored_data_output` - папка с ответами на оцениваемые тесты.

Пример использования чекера: `./check scored_data/vrp_421_41_1 scored_data_output/sc_421_41_1.out`

## Текущее решение

Сейчас для решения TSP написана динамика по подмножествам для размеров не более $20$ и жадник иначе. На практике все размеры не более $20$ и запускается это не очень часто, поэтому эта часть работает неплохо.

Разбиения на множества сейчас ищутся довольно тривиальные: бьется все жадно по углу. Работает это соответственно довольно плохо.
